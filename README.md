# FileRenamePs

ファイル名の長さ制限に対応するバッチリネームツール（PowerShell）。

## 機能

- **絵文字・特殊文字の削除**: サロゲートペア（絵文字等）、ゼロ幅文字、異体字セレクタなどを除去（削除後にファイル名が空になる場合は `renamed` にフォールバック）
- **フルパス 260文字制限**（Windows MAX_PATH 向け）: Rename-Itemが失敗しないよう先にチェック
- **ファイル名 255バイト制限**（Linux NAS / ext4 / XFS / Btrfs 向け）: UTF-8バイト数で計測し、超過分を末尾から切り詰め
- 拡張子は常に保持
- UTF-16サロゲートペアの分断を防止
- リネーム前に変更後のファイル名を提示し、Y/N で確認

## 使い方

```powershell
.\FileRename.ps1
```

起動時に対象ディレクトリのパスを入力するプロンプトが表示されます。
バイト上限・文字数上限はスクリプト冒頭で設定できます：

```powershell
$maxBytes = 255       # ファイル名のバイト上限（Linux NAS用）
$maxPathChars = 260   # フルパスの文字数上限（Windows用）
```

## 処理順序

1. 絵文字・特殊文字の削除
2. Windowsフルパス260文字制限のチェック・切り詰め
3. Linux NASファイル名255バイト制限のチェック・切り詰め
